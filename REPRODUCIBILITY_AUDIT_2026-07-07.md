# Reproducibility audit and cleanup plan

Date: 2026-07-07

## Executive summary

The repository contained a strong and reproducible small quantum-lattice core, but the later information-matter component reports were not backed by committed generator scripts or raw logs.

The cleanup action taken here is to separate:

```text
CODE_BACKED quantum/core simulations
from
PHENOMENOLOGY_NOTE information-matter design reports
```

This prevents Markdown-only numerical tables from being treated as reproducible results.

## Audit findings

### 1. Code-backed core is valid

The following scripts are committed and runnable:

```text
scripts/quantum_lattice_core.py
scripts/history_droplet_core.py
```

The code implements:

```text
EXCH / XY hopping
ZZ adjacent interaction phase
partial trace for two-qubit reduced density matrices
negativity via partial transpose negative eigenvalues
classical complex-wave control
history-droplet no-jump/postselective exploration
classical ablation/control logic
```

This layer remains the reproducible core of the repository.

### 2. Phenomenology reports are not yet reproducible

Most `results/*.md` files about information chemistry, reaction terrain, membrane, reservoir, source-sink, and material-property maps contain numerical tables without committed scripts, seeds, or raw logs.

They are therefore reclassified as:

```text
PHENOMENOLOGY_NOTE
```

They should not be cited as reproducible results until regenerated by committed scripts.

### 3. Phase-dependent transport is quarantined

The previously listed surviving claim:

```text
history-terrain phase-dependent transport
```

is not currently reproduced by the committed scripts. Audit notes indicate that the tested droplet and terrain did not overlap, so the claimed phase routing is not demonstrated.

It is reclassified as:

```text
QUARANTINED_CLAIM
```

### 4. Raw logs were blocked by .gitignore

The previous `.gitignore` ignored:

```text
results/*.json
results/*.csv
```

This conflicts with the stated goal of preserving raw logs. The ignore rules have been changed so raw json/csv logs may be committed.

## Promotion criteria

A result may be promoted from `PHENOMENOLOGY_NOTE` to `CODE_BACKED` only if it has:

```text
1. committed generator script
2. fixed seed/config
3. committed raw json/csv/jsonl log or equivalent
4. markdown report generated from that log
5. explicit classical/quantum control status
```
